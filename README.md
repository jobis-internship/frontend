# 자비스 11기 인턴십

## 브랜치 전략 (Trunk Based)

### Master 브랜치
- **main**: 직접 커밋 불가. pr을 통한 수정 가능.

### 작업 흐름
1. **기능 개발**
  - 이슈 생성
  - 브랜치 생성 후 이슈에 연결
  - 브랜치에서 작업
  - 작업 완료 후 main 브랜치로 pr

2. **PR 머지**
   - 최소 1명의 리뷰 후 머지
   - 머지 후 브랜치 삭제

---

## ✨ 커밋 컨벤션

### 커밋 메시지 형식
```
타입: 간결한 설명 (50자 이내)
```

### 커밋 타입
- **feat**: 새로운 기능
- **fix**: 버그 수정
- **refactor**: 코드 리팩토링
- **chore**: 빌드, 설정 파일 변경

### 예시
```
feat: 로그인 페이지 UI 구현
fix: 버튼 클릭 이벤트 오류 수정
refactor: API 호출 함수 모듈화
```
